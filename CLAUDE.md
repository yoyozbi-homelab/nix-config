## Project Overview

This is a personal NixOS configuration repository that manages both servers and desktop systems using Nix flakes. The configuration includes multiple hosts (servers and desktops) with automated deployment via GitHub Actions and Cachix.

## Architecture

### Host Configuration Structure

Every host is defined by a `hosts/<hostname>/host.toml` file. `flake.nix` reads these files via `lib/hosts.nix` and derives all flake outputs automatically — no hardcoded host lists anywhere.

- **NixOS hosts**: `nixosConfigurations` generated by `mkHostFromToml` (all hosts with `nixos = true`)
- **Home Manager configs**: `homeConfigurations` generated by `mkHomeFromToml` (all hosts with `home = true`)
- **Cachix-deploy agents**: `defaultPackage` built from server hosts (`nixos = true`, no `desktop`)

### Key Architecture Components

1. **TOML loader** (`lib/hosts.nix`):
   - Reads and validates every `hosts/*/host.toml`
   - Provides filtered views: `hosts.nixos`, `hosts.home`, `hosts.deploy`, `hosts.network`
   - `mkHostFromToml data` → `nixpkgs.lib.nixosSystem { ... }`
   - `mkHomeFromToml data` → `home-manager.lib.homeManagerConfiguration { ... }`

2. **Base NixOS module** (`nixos/core/default.nix`):
   - Imported by every host; sets overlays, nix settings, gc, sops, disko
   - `nixos/core/network-options.nix`: declares `networking.yoyozbi.hosts` options
   - Network data is injected from TOML via an inline module in `mkHostFromToml`

3. **Roles** (`nixos/roles/`, `home/roles/`):
   - Reusable NixOS modules selected per-host via the `roles` TOML array
   - Desktop roles live under `nixos/roles/desktop/`

4. **Per-host hardware** (`hosts/<hostname>/hardware.nix`):
   - Boot loader, kernel modules, filesystems, hardware-specific options
   - Imported automatically by `mkHostFromToml` when the file exists

### Host Types
- **Desktop hosts**: `laptop-nix` (KDE+Hyprland), `surface-nix` (GNOME), `vm-nix` (Noctalia/Hyprland)
- **Server hosts**: `ocr1` (K3s master/aarch64), `tiny1`/`tiny2` (K3s agents), `rp` (RPi4 K3s/aarch64)
- **Home-only hosts**: `wsl-nix`, `laptop-omarchy` (no NixOS, only `homeConfigurations`)

## host.toml Schema

All keys live under a `[host]` table. A `[network]` table is optional (servers only).

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `username` | string | **required** | Primary user |
| `platform` | string | `"x86_64-linux"` | `"x86_64-linux"` or `"aarch64-linux"` |
| `desktop` | string | `null` | Desktop selector (`kde`/`gnome`/`noctalia`/…); absent = server |
| `nixos` | bool | `true` | Generate a `nixosConfigurations` entry |
| `home` | bool | `false` | Generate a `homeConfigurations` entry |
| `build-home` | bool | `false` | Embed home-manager inside the NixOS build |
| `roles` | [string] | `[]` | NixOS role modules from `nixos/roles/` (supports `"desktop/hyprland"` paths) |
| `hardware` | [string] | `[]` | `inputs.nixos-hardware.nixosModules.<name>` entries |
| `state-version` | string | flake default | NixOS/HM state version |

`[network]` keys (servers): `internalIp`, `externalIp`, `mac`, `rancher`, plus sub-tables `traefik-dashboard`, `portainer`, `argocd`, `flux`, `longhorn` each with `{enabled, dashboardUrl}`.

## Adding a New Host

```bash
# 1. Create the host directory
mkdir hosts/newhost

# 2. Write the TOML declaration
cat > hosts/newhost/host.toml << 'EOF'
[host]
username = "nix"
platform = "x86_64-linux"
roles = ["boot-grub", "cachix", "openssh", "networkmanager"]

[network]
internalIp = "10.0.0.X"
externalIp  = "1.2.3.4"
mac         = "aa:bb:cc:dd:ee:ff"
EOF

# 3. Write hardware config (boot, filesystems, kernel modules)
$EDITOR hosts/newhost/hardware.nix

# 4. Verify it evaluates
nix eval .#nixosConfigurations.newhost.config.system.build.toplevel.drvPath

# 5. For aarch64 hosts (can't build locally), eval only:
nix eval .#nixosConfigurations.newhost.config.system.build.toplevel.drvPath
```

## Common Development Commands

### Building and Testing
```bash
# Build specific host configuration
nixos-rebuild build --flake .#hostname

# Build and switch (requires root)
sudo nixos-rebuild switch --flake .#hostname

# Build home-manager configuration
home-manager switch --flake .#username@hostname

# Build VM (for vm-nix — use this, not toplevel)
nix build .#nixosConfigurations.vm-nix.config.system.build.vm

# Update flake inputs
nix flake update
```

### Remote Deployment
```bash
# Deploy to remote host using nixos-rebuild
nixos-rebuild --target-host root@hostname --flake .#hostname switch

# Deploy using deploy-rs
nix run github:serokell/deploy-rs -- .#hostname
```

### Secret Management (SOPS)
```bash
# Edit encrypted secrets
nix-shell -p sops --run "sops hosts/rp/rp-sec.yml"
nix-shell -p sops --run "sops nixos/roles/ocr-cluster/ocr-secrets.yml"

# Update keys for existing secrets after adding new hosts
nix-shell -p sops --run "sops updatekeys nixos/roles/ocr-cluster/ocr-secrets.yml"

# Get age key from SSH host key
nix-shell -p ssh-to-age --run 'ssh-keyscan <hostname> | ssh-to-age'
```

### Server-Specific Commands
```bash
# Build SD card image for Raspberry Pi
nix run nixpkgs#nixos-generators -- -f sd-aarch64 --flake .#rp --system aarch64-linux -o ../pi.sd

# Reset K3s node (on server)
./nixos/roles/k3s-server/k3s-reset-node
```

## File Structure

```
hosts/
  <hostname>/
    host.toml        # TOML declaration (required)
    hardware.nix     # boot/fs/kernel config (auto-imported if present)
    disks.nix        # disko disk layout (imported from hardware.nix)
    *-sec.yml        # SOPS secrets (referenced from hardware.nix)
nixos/
  core/
    default.nix          # base module imported by every host
    network-options.nix  # networking.yoyozbi.hosts option declarations
  roles/               # reusable NixOS modules (role names in host.toml)
    desktop/           # desktop environment roles
  users/               # per-user NixOS modules (root, nix, yohan)
home/                  # home-manager configuration
  default.nix          # entry point for all home configs
  users/yohan/         # user-specific home modules
  desktops/            # per-DE home modules
lib/
  default.nix          # exports mkHostFromToml, mkHomeFromToml, hosts
  hosts.nix            # TOML loader, validators, generators
pkgs/                  # custom packages
overlays/              # nixpkgs overlays
```

## Development Notes

- State version is centrally managed in `lib/hosts.nix` (currently "25.05")
- Uses unstable Nix package for latest features
- Cachix deployment configured for automated updates
- All unfree packages allowed in nixpkgs config
- Custom overlays defined in `overlays/`
- `vm-nix` is VM-only: its `toplevel` intentionally fails (no real filesystems); use `system.build.vm`
